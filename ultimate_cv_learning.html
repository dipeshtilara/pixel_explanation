What Exactly Is a Pixel?

A pixel (picture element) is the smallest unit of a digital image — a tiny coloured square storing exactly three numbers: Red, Green, Blue, each from 0 to 255. Drag the sliders and see what colour you mix.

That's all a pixel is.

Colour mixing rules

R=255, G=0, B=0 → 🔴 Red R=0, G=255, B=0 → 🟢 Green R=0, G=0, B=255 → 🔵 Blue R=255, G=255, B=0 → 🟡 Yellow R=255, G=255, B=255 → ⬜ White R=0, G=0, B=0 → ⬛ Black

Memory cost

1 channel → 1 byte (0–255) 1 pixel → 3 bytes (RGB) 1 MP image → ~3 MB (raw) 12 MP phone → ~36 MB (raw)
💡

Key insight: 256 × 256 × 256 = 16,777,216 possible colours from just three numbers. Pixels are not pictures — they are instructions.

Three Types of Digital Images

Not all images use RGB. Each type stores pixels differently — and each exists because specific applications need exactly that format.

Binary Image — 1 bit per pixel

Each pixel is either 0 (black) or 1 (white). Fastest to process, smallest file size. Used where colour is irrelevant.

Pixel ∈ {0, 1} Bits per pixel: 1 Memory (8×8): 8 bytes Use cases: • OCR document scanning • Barcode / QR reading • Edge detection output • Fingerprint matching

Each pixel is a single brightness value 0–255. No colour, just luminance. 3× smaller than RGB.

Module 1 · Concept 03

An Image Is a Grid of Pixels

Zoom into any digital photo and you will see individual coloured squares. An image is simply a rectangular grid — more squares means more detail. Less squares means blockier, blurrier pictures.

Resolution demonstration — drag the slider

See how resolution affects image quality. Same information, different grid sizes.

Resolution: 4 × 4 = 16 pixels Memory: 16 × 3 = 48 bytes Detail: Very low (pixelated)
📐

More pixels = more detail but larger file size. A 4K video frame (3840×2160) contains 8,294,400 pixels — each needing 3 bytes = 24 MB per single frame.

🎯

Real numbers: A 12 megapixel smartphone photo has 12,000,000 pixels. At 3 bytes each that is 36 MB raw before any compression. JPEG reduces this to ~3–5 MB by removing detail the eye cannot see.

Module 1 · Concept 04

The Pixel Matrix — Images Are Arrays

Every image is stored as a 2D (or 3D) array of numbers. In Python and OpenCV, this is a NumPy array. Understanding this is the key that unlocks all of computer vision.

A 6×6 pixel grid — this IS an image

Each cell is a brightness value (0 = black, 255 = white). Numbers form shapes.

import numpy as np
import cv2

# An image is just a NumPy array
blank = np.zeros((300, 300, 3), dtype=np.uint8)

# Read a single pixel's BGR values
pixel = frame[100, 150]   # row, column
print("BGR:", pixel)   # [120, 45, 200]

# Shape tells you dimensions
print(frame.shape)  # (480, 640, 3)
#                     H    W   channels

# Crop a region (Region of Interest)
roi = frame[50:200, 80:300]

Array shapes by image type

Binary: [H, W] → 2D, 1 value Grayscale: [H, W] → 2D, 1 value Colour: [H, W, 3] → 3D, 3 values Example 640×480 image: Grayscale: 307,200 values Colour: 921,600 values (3×)
📚

Class 12 — NumPy: frame.shape returns (height, width, channels). You can slice: frame[y1:y2, x1:x2] crops a Region of Interest. Brighten = frame + 50. Flip = np.flip(frame, axis=1). Images are just maths on arrays.

b_ch, g_ch, r_ch = cv2.split(img)

⚠️

OpenCV stores BGR, not RGB! When you load an image with cv2.imread(), the channel order is Blue-Green-Red. This is the #1 mistake beginners make — red and blue appear swapped. Always convert with cv2.cvtColor(img, cv2.COLOR_BGR2RGB) before displaying with matplotlib.

Image Reconstruction — Pixels → Picture

Every image is reconstructed the same way: read the matrix row by row, paint each pixel its stored colour. This animation shows the process live — watch numbers become a picture.

Live reconstruction animation

A 24×24 pixel face is reconstructed from its numeric matrix. Watch it build pixel by pixel.

Status: Click ▶ to begin Row: 0 / 24 Col: 0 / 24 Pixels: 0 / 576
🔁

Same pixel data always = same image. The matrix is the image. Reading it back reproduces the picture perfectly every time.

# Load image and access its data
img = cv2.imread('photo.jpg')
print(img.shape)  # (H, W, 3) — the matrix dimensions

# Read row by row — this IS reconstruction
for row in range(img.shape[0]):           # each row
    for col in range(img.shape[1]):       # each column
        b, g, r = img[row, col]           # read pixel
        # paint pixel (r, g, b) at position (col, row)

# Save reconstructed image
cv2.imwrite('reconstructed.jpg', img)
# Same pixel data = identical image every time
💡

Key takeaway: Pixels → Numbers → Matrix → Numbers → Pixels. The matrix IS the image. Digital images are not "pictures" stored somewhere — they are instructions: "paint pixel at position (x,y) with colour (R,G,B)", repeated millions of times.

Module 2 · Concept 01

Digital Eyes: Capturing the Frame Stream

Video is not a continuous stream — it is many still photographs shown very fast (24–60 per second). Each captured still is called a frame. Our while loop grabs one frame at a time.

🎯

Key concept: cap.read() captures ONE frame — like pressing the shutter button on a camera. We call it thousands of times per minute inside a loop. Each frame is a NumPy array ready for processing.

import cv2

# Connect to webcam (0 = built-in, 1 = external)
cap = cv2.VideoCapture(0)

while True:
    # Grab one frame — like one photograph
    ret, frame = cap.read()

    if not ret:           # camera failed?
        break

    # Show the frame in a window
    cv2.imshow('Live Stream', frame)

    # 1ms wait — press Q to quit
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()          # free the camera
cv2.destroyAllWindows()

Line-by-line explanation

VideoCapture(0)Connects to camera index 0. Try index 1 if two cameras exist.
cap.read()Returns ret (did it work?) and frame (image as grid of numbers).
waitKey(1)Without this the window freezes. Gives OS 1ms to refresh display and listens for keys.
cap.release()Frees the camera so other apps can use it. Always include this — like hanging up the phone.
● FRAME 000

Webcam frame stream simulation

✅

Expected output: A window opens showing your webcam feed. The frame counter increases ~30× per second. Press Q to close cleanly.

🔴

Common error: If nothing opens, try VideoCapture(1). If that fails, check that no other app (Zoom, Teams) is using the camera.

Simplification: The Grayscale Shift

Colour is expensive. Each colour pixel is THREE numbers. Converting to grayscale reduces data by 66%, making every algorithm roughly 3× faster — with almost no loss in useful information for most CV tasks.

🎯

Key concept: OpenCV stores colour in BGR order (Blue, Green, Red — not RGB!). Grayscale collapses the three channels into one brightness value using a weighted formula.

ITU-R BT.601 formula
Y = 0.114×B + 0.587×G + 0.299×R
Result: — / 255

gray_frame = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

# Show both windows side by side
cv2.imshow('Original Colour', frame)
cv2.imshow('Grayscale', gray_frame)
📐

Class 12 deeper dive: The weights 0.299 (R), 0.587 (G), 0.114 (B) are NOT equal because the human eye is most sensitive to green light. These are the ITU-R BT.601 coefficients used in broadcast TV since the 1980s — the same standard in every phone camera today.

The Matrix Behind the Magic

An image IS a matrix. Every visual operation is just maths on numbers. Flip an image = reverse an array. Brighten = add a constant. Edge detect = apply a filter matrix (convolution).

Thresholding simulator — drag the slider

Thresholding converts a grayscale image into binary: pixels above the threshold become white (255), below become black (0).

127

Each circle = one pixel. White = above threshold (255). Dark = below threshold (0).

pixel = frame[100, 150]  # row 100, col 150

Module 2 · Concept 04

Finding Boundaries: Canny Edge Detection

Edges are where brightness changes sharply between neighbouring pixels. By finding sharp changes, we can outline every object without knowing what those objects are.

What is a gradient? — spot the edge

The green column is where the difference between neighbours is largest — that is the edge.

Canny's 4-step process

1

Gaussian blur

Slightly blur the image first to remove random camera noise, which can look like fake edges.

2

Gradient calculation (Sobel)

For every pixel, compute how much brightness changes in X and Y directions. High change = potential edge.

3

Non-maximum suppression

Thin out wide edges. Keep only pixels that are the local peak in their gradient direction.

4

Double threshold + hysteresis

Strong edges (above high threshold) are kept. Weak edges kept only if connected to strong ones.

# Canny(blurred, 150, 250)  →  fewer, cleaner edges
🚗

Real-world application: Tesla Autopilot and Waymo use edge detection as a first step to identify lane markings, pedestrian outlines, and obstacles before deeper neural network processing.

📐

Class 12 maths: Gradient magnitude: G = √(Gx² + Gy²) where Gx and Gy are convolutions with the Sobel operator — a direct application of partial derivatives to a 2D matrix.

Module 2 · Concept 05

Object Intelligence: Face & Eye Detection

The machine is not "thinking" — it is pattern matching. A tiny window slides across your image thousands of times, checking whether pixel patterns match a pre-learned face recipe (a Haar Cascade).

What is a Haar Cascade?

Invented by Viola and Jones (2001). Faces share universal patterns: eye sockets are darker than the nose bridge, the forehead lighter than the eyebrows. These rectangular brightness-difference patterns (Haar features) are compared millions of times across the image.

Sliding window — minNeighbors simulator

5
import cv2

face_cascade = cv2.CascadeClassifier(
    cv2.data.haarcascades + 'haarcascade_frontalface_default.xml')
eye_cascade  = cv2.CascadeClassifier(
    cv2.data.haarcascades + 'haarcascade_eye.xml')

cap = cv2.VideoCapture(0)
while True:
    ret, frame = cap.read()
    if not ret: break
    gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

    faces = face_cascade.detectMultiScale(gray, 1.3, 5)
    for (x,y,w,h) in faces:
        cv2.rectangle(frame,(x,y),(x+w,y+h),(255,0,0),2)

        roi_gray  = gray[y:y+h, x:x+w]    # search eyes in face only
        roi_color = frame[y:y+h, x:x+w]
        eyes = eye_cascade.detectMultiScale(roi_gray)
        for (ex,ey,ew,eh) in eyes:
            cv2.rectangle(roi_color,(ex,ey),(ex+ew,ey+eh),(0,255,0),2)

    cv2.imshow('Detection', frame)
    if cv2.waitKey(1) & 0xFF == ord('q'): break

cap.release(); cv2.destroyAllWindows()
📚

Class 12 — limitations: Haar cascades fail on tilted faces, low light, and non-frontal angles. Modern systems use CNNs (Convolutional Neural Networks) that learn their own features automatically. That is your gateway to deep learning — the next level.

Deep Learning: MobileNetV2 Image Classification

Skip training entirely. MobileNetV2 was already trained on 1.2 million images across 1000 classes. Use that knowledge directly to classify your own photos in seconds.

from tensorflow.keras.applications import MobileNetV2
from tensorflow.keras.applications.mobilenet_v2 import preprocess_input, decode_predictions
from tensorflow.keras.preprocessing import image
import numpy as np

model = MobileNetV2(weights='imagenet')  # downloads ~14 MB once

img  = image.load_img('photo.jpg', target_size=(224,224))
x    = image.img_to_array(img)
x    = np.expand_dims(x, 0)
x    = preprocess_input(x)    # CRITICAL — normalise to [-1, 1]

preds = model.predict(x)
for lbl, name, prob in decode_predictions(preds, top=3)[0]:
    print(f"{name}: {prob:.1%}")  # e.g. "tabby_cat: 82.4%"
⚠️

Critical: preprocess_input() normalises pixel values from [0,255] to [-1,1], which is the range MobileNetV2 was trained with. Skipping this gives completely meaningless predictions. This one line matters more than it looks.

Transfer learning — freeze and train

10
📚

What is transfer learning? MobileNetV2 learned to recognise edges, textures, and shapes from 1.2 million images. Freeze those 154 layers (base.trainable = False). Add 2 new layers and train only those on your data. Result: a custom classifier trained in minutes on a CPU, not days on a GPU.
